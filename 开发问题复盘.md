# Var课程表同步器开发问题复盘

本文记录本次插件开发过程中遇到的主要问题、根因分析、解决办法与后续建议，便于后续维护和二次开发快速定位。

## 1. UI 按钮点击无反应

### 现象
- 页面能渲染，但所有按钮点击后无任何反馈。
- 初期无法确认是“事件没进插件”还是“事件进来后没命中逻辑”。

### 根因
- 事件处理链路与实际宿主事件行为不完全一致。
- 事件处理逻辑初期写法偏离示例项目风格，排查难度高。
- 仅依赖单一事件类型（如只处理 `Click`）时，某些端上可能无法覆盖全部点击行为。

### 解决办法
- 将逻辑层改为示例项目风格：
  - `on_ui_render` 只负责渲染入口；
  - `on_ui_event` 负责分发；
  - `ui_event_processor` 统一处理事件。
- 为关键按钮补充点击类事件绑定（`Click` + `PointerUp`）。
- 增加事件可视化调试信息（记录最近事件），用于判断事件是否进入插件。

### 结果
- 按钮事件链路恢复，交互可用。

## 2. Rust 模块与 UI API 兼容性问题

### 现象
- 出现语法错误（如 `pub mod ui as plugin_ui;`）。
- 出现大量 UI 方法不存在错误（如 `justify_content`、`flex_grow`、`text_align`）。

### 根因
- 使用了 Rust 不支持的模块别名声明方式。
- 使用了当前 WIT 绑定中不存在的 UI API（文档示例与本地 WIT 存在差异）。

### 解决办法
- 模块组织改为标准 Rust 写法（后续进一步重构为 `src/ui/*` 模块目录）。
- UI 调用全部以当前项目 `wit/deps/astrobox-psys-host.wit` 为准，不再假设文档中的旧接口可用。
- 对齐示例项目的链式 UI 与事件处理风格。

### 结果
- 编译恢复稳定，UI 构建链路可维护。

## 3. 推送/获取失败：Quick app not found

### 现象
- 日志报错：`Failed to send QAIC message ... Quick app com.azuma.syclass not found ...`
- 设备已连接，权限也已通过，但通信失败。

### 根因
- 写死包名 `com.azuma.syclass`，与手表实际安装快应用包名不一致。

### 解决办法
- 发送前动态查询手表应用列表（`thirdpartyapp::get_thirdparty_app_list`）。
- 优先按应用名关键字匹配（`Var课程表`），再按候选包名回退匹配。
- 匹配成功后再执行：
  - `register_interconnect_recv`
  - `interconnect::send_qaic_message`

### 结果
- 解决“权限正常但通信失败”的核心问题，推送/获取恢复可用。

## 4. 回调处理异常：手环已回数据但插件仍“等待”

### 现象
- 手环端已返回课程数据，但插件状态仍停留在等待或请求已发送。

### 根因
- 回调解析过于严格，早期仅识别单一结构（例如固定 `type + classes`）。
- 实际回包可能存在多层包裹字段，或字段命名差异。

### 解决办法
- 放宽回包解析策略：
  - 允许多层递归解包；
  - 扩展常见包裹字段（如 `payloadText`、`content`、`body`、`result` 等）；
  - 支持多种课程字段别名（如 `name/courseName`、`start/startSection`、`end/endSection`）。
- 解析失败时输出回包摘要到状态文案，提升可观测性。

### 结果
- 回调命中率显著提升，能正确入库并刷新 UI。

---

## 5. 架构层面调整（本次“魔改示例项目”）

### 调整内容
- 将 UI 从单文件重构为示例风格模块化结构：
  - `src/ui/mod.rs`
  - `src/ui/state.rs`
  - `src/ui/build.rs`
  - `src/ui/event_handler.rs`
  - `src/ui/device.rs`
- `lib.rs` 只保留生命周期与事件入口分发，避免业务与渲染耦合。

### 收益
- 结构更接近示例，维护成本更低。
- 事件、通信、状态、渲染职责更清晰。

## 7. 后续建议

1. 固化“接口真相源”
- 统一以项目 `wit/deps/*.wit` 为准，文档仅作参考。

2. 增加回归测试清单（手工）
- 按钮点击、获取、回调入库、编辑、删除、推送全链路走一遍。
- 覆盖“包名不匹配”“回包结构变化”两个高风险场景。

3. 保留最小诊断开关
- 保留事件摘要与回包摘要能力（可通过开关控制显示），便于现场排障。

4. 如需进一步稳定
- 可增加“启动目标快应用”按钮（调用 `thirdpartyapp::launch_qa`）用于用户自助修复“快应用未运行/未安装”场景。
